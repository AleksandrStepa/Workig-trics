--ликбез по Git
1. Клонируешь репозиторий
2. Заходишь в репозиторий локально, там уже есть git
3. git status - посмотреть в какой ветке находишься
4. git remote -v можно помотреть какой remote есть и как он называется
5. git checkout -b main создаешь свою ветку, куда копируется весь мастер (название может быть любое)
6. вносишь изменения
7. git add . добавляешь изменения для отслеживания
8. git commit -m "что-нибудь" комитишь изменения
9. git push origin main пушишь уже свою ветку в проект (например для ревью)
10. git checkout master - переходишь локально опять в master
11. git merge main мержишь с вою ветку в мастер, тоже локально
12. git push origin master пушишь локально измененный мастер в удаленный репозиторий, но уже в мастер
git log проверить, что все прошло нормально
